applied.fm Style Guide for Coq

About

This is a work in progress! We welcome feedback and pull requests. Find us on GitHub.

Principles

Use cases

  1. User/contributor/curious person browsing the GitHub repository in a browser
  2. User/contributor/curious person exploring the code in a proof assistant
  3. User/contributor who needs to isolate and resolve build issues
  4. User trying to explore undocumented features
  5. Researcher trying to understand the architecture
  6. Contributor trying to ship a new feature
  7. Contributor engaged in a large refactor

Objectives

  1. Find & replace work as expected.
  2. Mitigate the law of triviality.
  3. Code is robust against changes to the Coq engine.
  4. Code is minimally impacted by breaking changes in upstream components.
